# Система релизов для StrodService

## Описание

Проект теперь использует систему тегов и релизов GitHub для управления версиями. Вместо автоматической сборки при каждом пуше в ветку `main`, теперь создаются полноценные релизы при создании тегов.

## Создание нового релиза

### Метод 1: С помощью Git тегов

1. Увеличьте номер версии в соответствующих местах проекта (если применимо)
2. Создайте коммит с изменениями:
   ```bash
   git add .
   git commit -m "Release version X.Y.Z"
   ```
3. Создайте тег:
   ```bash
   git tag vX.Y.Z
   ```
4. Отправьте изменения и тег на GitHub:
   ```bash
   git push origin main
   git push origin vX.Y.Z
   ```

### Метод 2: Через интерфейс GitHub

1. Перейдите на вкладку "Releases" в репозитории
2. Нажмите "Draft a new release"
3. Выберите или создайте новый тег в формате `vX.Y.Z`
4. Укажите название и описание релиза
5. Нажмите "Publish release"

## Что входит в полноценный релиз

Новый workflow `release-on-tag.yml` включает в себя:

1. **Кроссплатформенную сборку**: Приложение собирается для Windows, Linux и macOS
2. **Python версия**: Сборка с помощью PyInstaller в исполняемые файлы
3. **Electron версия**: Сборка десктопного приложения с графическим интерфейсом
4. **Исходный код**: Архив с исходным кодом проекта
5. **Все компоненты**: Backend (Python/FastAPI), Frontend, документация и другие модули

## Файлы, включенные в релиз

Для каждой платформы будут собраны:

- Исполняемые файлы приложения
- Зависимости приложения
- База данных (если требуется)
- Конфигурационные файлы

## Структура версий

Версии следуют формату Semantic Versioning (SemVer): `vX.Y.Z`
- `X` - мажорная версия (крупные изменения)
- `Y` - минорная версия (новые функции)
- `Z` - патч версия (исправления багов)

## Автоматизация

GitHub Actions автоматически:
- Обнаруживает новые теги, начинающиеся с `v`
- Запускает процесс сборки для всех поддерживаемых платформ
- Создает релиз на GitHub с собранными артефактами
- Загружает все собранные файлы как ассеты релиза
