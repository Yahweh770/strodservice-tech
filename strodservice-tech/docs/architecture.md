# Архитектура приложения "Строд-Сервис Технолоджи"

## Общая архитектура
Приложение представляет собой веб-систему управления строительными проектами с модульной архитектурой, состоящей из нескольких основных компонентов:

1. **Frontend** - пользовательский интерфейс на React с TypeScript
2. **Backend-Python** - API и бизнес-логика на Python (FastAPI)
3. **Backend-DotNet** - альтернативная реализация API и бизнес-логики на .NET 8
4. **База данных** - PostgreSQL для хранения структурированных данных
5. **Файловое хранилище** - для документов, схем и отчетов
6. **Система уведомлений** - для контроля остатков материалов

## Структура данных

### Проекты
- ID
- Название
- Описание
- Дата подписания контракта
- Дата укладки асфальта
- Дата начала работ
- Дата завершения работ
- Статус
- Дополнительные комментарии

### Материалы
- ID
- Название
- Номер партии
- Дата изготовления
- Срок годности
- Количество
- Единицы измерения
- Сертификаты и документация

### Исполнительная документация
- ID
- Ссылка на проект
- Тип документа (АООК/АОСР)
- Ссылка на файл
- Дата создания
- Комментарии

## Микросервисы
Приложение может быть разделено на несколько микросервисов:
- Управление проектами
- Управление материалами
- Документооборот
- Уведомления
- Файловое хранилище

## Безопасность
- Аутентификация через JWT токены
- Авторизация по ролям (инженер ПТО, менеджер, администратор)
- Защита от несанкционированного доступа к документам
