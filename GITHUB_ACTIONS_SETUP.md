# Настройка автоматической сборки релизов через GitHub Actions

## Описание

В проекте настроена автоматическая сборка .exe файла при каждом пуше в ветку `main` с помощью GitHub Actions.

## Конфигурация

Файл `.github/workflows/release-on-push.yml` содержит следующую конфигурацию:

- Запускается автоматически при пуше в ветку `main`
- Использует скрипт `build-pyinstaller.sh` для сборки проекта
- Создает исполняемый файл с помощью PyInstaller
- Генерирует релиз с автоматически увеличивающимся номером версии
- Загружает .exe файл как ассет релиза

## Структура проекта

- `main.py` - основной входной файл приложения
- `doc_tracking_system.py` - реализация системы учета документов
- `build-pyinstaller.sh` - скрипт для сборки .exe файла
- `requirements.txt` - зависимости проекта

## Как активировать функциональность

1. Убедитесь, что файл `.github/workflows/release-on-push.yml` присутствует в репозитории
2. Закоммитьте изменения в ветку `main`
3. После каждого пуsha в ветку `main` будет автоматически создаваться новый релиз с .exe файлом

## Результаты сборки

Собранный .exe файл будет доступен в разделе Releases вашего репозитория GitHub как ассет релиза.

## Тестирование локально

Для тестирования локальной сборки выполните:

```bash
chmod +x build-pyinstaller.sh
./build-pyinstaller.sh
```

Собранный файл будет находиться в папке `dist/`.
