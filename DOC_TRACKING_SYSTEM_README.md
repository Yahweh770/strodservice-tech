# Система учета отправленных и возвращенных документов исполнительной документации в ПТО

## Описание

Система предназначена для учета отправленных и возвращенных документов исполнительной документации в производственно-техническое объединение (ПТО). Приложение позволяет:

- Добавлять документы в систему
- Фиксировать отправку документов
- Регистрировать возврат документов
- Просматривать статус документов
- Осуществлять поиск документов

## Архитектура

Система состоит из следующих компонентов:

### База данных (SQLite)

Содержит три основные таблицы:

1. **documents** - основная информация о документах:
   - id: уникальный идентификатор
   - doc_number: номер документа
   - doc_title: наименование документа
   - project_number: номер проекта
   - issue_date: дата выпуска
   - doc_type: тип документа
   - status: статус документа ('в наличии', 'отправлен', 'возвращен')
   - created_at: дата создания записи

2. **shipments** - информация об отправках:
   - id: уникальный идентификатор
   - doc_id: ссылка на документ
   - recipient: получатель
   - shipment_date: дата отправки
   - notes: примечания

3. **returns** - информация о возвратах:
   - id: уникальный идентификатор
   - doc_id: ссылка на документ
   - return_date: дата возврата
   - condition: состояние документа
   - notes: примечания

## Установка и запуск

Система написана на Python и использует встроенные библиотеки, поэтому дополнительная установка зависимостей не требуется.

### Требования

- Python 3.6 или выше
- Модули: sqlite3, datetime, os (входят в стандартную библиотеку Python)

### Запуск

```bash
python3 doc_tracking_system.py
```

## Использование

После запуска программы откроется интерактивное меню с возможностями:

1. **Добавить документ**
   - Введите номер документа
   - Введите наименование документа
   - При необходимости укажите номер проекта, дату выпуска и тип документа

2. **Отправить документ**
   - Укажите ID документа для отправки
   - Введите получателя
   - При необходимости укажите дату отправки и примечания

3. **Зафиксировать возврат документа**
   - Укажите ID документа для возврата
   - При необходимости укажите состояние документа и примечания

4. **Просмотреть все документы**
   - Отображает полный список документов с их текущим статусом, датами отправки и возврата

5. **Найти документ**
   - Поиск документов по номеру, наименованию или номеру проекта

6. **Выход**
   - Завершение работы программы

## Пример использования

После запуска программы:

```
=== Система учета документов ПТО ===
1. Добавить документ
2. Отправить документ
3. Зафиксировать возврат документа
4. Просмотреть все документы
5. Найти документ
6. Выход
Выберите действие (1-6):
```

## Статусы документов

- **в наличии**: документ находится в офисе ПТО
- **отправлен**: документ отправлен (после выполнения операции отправки)
- **возвращен**: документ возвращен (после выполнения операции возврата)

## Файл базы данных

База данных создается автоматически при первом запуске программы и сохраняется в файл `pto_docs.db` в директории запуска.

## Безопасность и резервное копирование

- Файл базы данных следует регулярно резервировать
- Для многопользовательского доступа рекомендуется мигрировать на серверную СУБД (PostgreSQL, MySQL)
